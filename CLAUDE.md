# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Mualim — a phone-first PWA for studying the HKSI LE Paper 1 exam in 3–4 question micro-sessions, with spaced repetition and a timed mock-exam mode. Live at https://muqaddaszehni.github.io/mualim/ (GitHub Pages, deploys from `main`). Design spec: `docs/superpowers/specs/2026-06-10-mualim-design.md`. Implementation plan (as built): `docs/superpowers/plans/2026-06-10-mualim.md`.

## Commands

```bash
npm test                                  # full vitest suite (29 tests, node env)
npx vitest run tests/scheduler.test.ts    # single test file
npm run dev                               # dev server at http://localhost:5173/mualim/
npm run build                             # tsc --noEmit + vite build (PWA sw.js emitted)
npm run preview                           # serve dist/ at http://localhost:4173/mualim/
```

Content pipeline (one-off; inputs are gitignored — see below):

```bash
python3 scripts/extract-topics.py         # PDF → content-src/t{N}-c{K}.txt + tasks.json (needs "P1 v3.5 eng full.pdf" in repo root and pdftotext/poppler)
node scripts/build-pack.mjs               # content-gen/*.json → public/packs/hksi-paper1/{questions,flashcards}.json
python3 scripts/make-icons.py             # regenerate PWA icons
```

Deploy = push to `main`; `.github/workflows/deploy.yml` runs test + build and publishes `dist/` to Pages. The repo name `mualim` must match `base: '/mualim/'` in `vite.config.ts`.

## Architecture: engine / pack / ui

The deliberate core split — a future "examify anything" platform:

- **`src/engine/`** — content-agnostic study logic. NO HKSI-specific values may appear here; every exam rule (question count, time limit, pass mark, topic weights, page ranges) is read from the pack manifest at runtime. Pure functions, fully unit-tested, no React.
  - `scheduler.ts` — compressed Leitner: boxes 0–4, due intervals [1h, 4h, 24h, 72h, 168h]; correct → box+1 (cap 4), wrong → box 0 (re-queued within the hour). Tuned for a 3-week exam sprint; don't "fix" the short intervals.
  - `sessionBuilder.ts` — builds each 4-item sit. Priority: due reviews (soonest first) → final-week mode (exam < 7 days away: weakest seen items) → new items ranked by topicWeight × unseen-fraction. Hard constraint: never a single-topic sit; soft target: 3 MCQ + 1 flashcard.
  - `mockExam.ts` — largest-remainder allocation over topic weights (tie-break: larger fraction, then larger weight, then lower id). `tests/mockExam.test.ts` pins the official 60-question distribution `{1:2,2:2,3:8,4:14,5:11,6:10,7:6,8:4,9:3}` — if allocation changes and that test fails, the change is wrong, not the test. `mulberry32` provides seeded determinism.
  - `storage.ts` — AppState in localStorage key `mualim:v1`, injectable storage for tests (node env has no localStorage). Streak uses LOCAL calendar days (deliberate fix; user is in HK, UTC+8).
  - `pack.ts` — `validatePack` is the schema gate for LLM-generated JSON; it is deliberately defensive against malformed runtime input (non-array options, missing type/id, non-number pageRef). Keep it paranoid.
- **`public/packs/hksi-paper1/`** — the content pack: `manifest.json` (exam rules + 9 topics with official weighting-midpoints summing to exactly 100, and verified PDF page ranges), `questions.json` (700 MCQs), `flashcards.json` (250). A new exam = a new pack folder, zero engine changes.
- **`src/ui/`** — thin React views over the engine. Hash routing in `App.tsx` (no router dep); all state mutations flow through engine functions and persist via `saveState` inside the `update()` wrapper. `Session` is rebuilt per mount and remounted via `key={sitKey}` for "Another sit". Dark-focus theme in `theme.css` (user-selected design; palette vars at top).

`tests/pack.content.test.ts` runs against the REAL committed pack (skips if absent): zero `validatePack` errors, per-topic counts ≥95% of targets, enough MCQs per topic for the mock allocation, >95% explanations cite a PDF page.

## Content provenance (anti-hallucination contract)

Every item was generated by an agent reading ONLY extracted study-guide text, then 100% fact-checked by an independent verifier agent (separate context), with failed items regenerated or dropped — never patched. A 10-question random re-verification scored 10/10. **If you add or edit items, preserve this contract**: regenerate from source text in `content-src/` and re-verify against it; never write exam content from model memory. Item ids follow `t{topic}-c{chunk}-q{n}` / `-f{n}` (`r` suffix = repaired replacement).

## Gitignored on purpose — do not commit

- `P1 v3.5 eng full.pdf` (HKSI copyrighted source; the repo is public) — and any other `*.pdf`
- `content-src/` (extracted PDF text), `content-gen/` (raw generated items)

This means the content pipeline cannot run from a fresh clone without the PDF. That's accepted; the committed pack in `public/packs/` is the source of truth for the app.
